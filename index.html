    Introduction

    Welcome to the Insurance Backend API documentation where you'll learn more about the endpoints available and how to use them including examples and expected responses.

    It is recommended that you use Postman, a great tool to quickly make requests to a local server you may have running.

    Authentication

    To authorize, use this code:

    // Login to the API server by making a GET request like the following:

GET 'http://localhost:8000/v1/user/login?user=manningblankenship@quotezart.com'

    Make sure to replace manningblankenship@quotezart.com with your user email which should be in the list of clients from this https://www.mocky.io/v2/5808862710000087232b75ac API. Once you're logged in, you'll see the following response:

    {
    "ok": true,
    "msg": "User logged in successfully",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOiJlOGZkMTU5Yi01N2M0LTRkMzYtOWJkNy1hNTljYTEzMDU3YmIiLCJpYXQiOjE1NjY3Mzg5MjZ9.F5vyQ-oSvxf8nkFZt3bklkIPQpFHoaIT59yvu9k-7M0"
}

    If you pass an invalid user you'll get the following response:

    {
    "ok": false,
    "msg": "Could not find that user email"
}

    To make requests, you first have to authenticate to the server by making a GET request to the /v1/user/login endpoint and passing your user email as a user query parameter.

    The server will then store a unique JWT token associated with your logged in user. You don't have to pass the token since it's already stored in the server for the entire session.

    Users

    Get a User by User ID

    GET 'http://localhost:8000/v1/user?id=e8fd159b-57c4-4d36-9bd7-a59ca13057bb'

    The above command returns JSON structured like this:

    {
    "ok": true,
    "msg": "User found successfully",
    "user": {
        "id": "e8fd159b-57c4-4d36-9bd7-a59ca13057bb",
        "name": "Manning",
        "email": "manningblankenship@quotezart.com",
        "role": "admin"
    }
}

    This endpoint finds and returns a user object given a user ID as the query parameter.

    Example HTTP Request

    GET http://localhost:8000/v1/user?id=e8fd159b-57c4-4d36-9bd7-a59ca13057bb

    Query Parameters

    Parameter Default Description
    id null The user ID must be defined or you'll get an invalid response

    Get a User by User Name

    GET 'http://localhost:8000/v1/user?name=Hatfield'

    The above command returns JSON structured like this:

    {
    "ok": true,
    "msg": "User found successfully",
    "user": {
        "id": "ce295158-0f59-41ad-b09e-e1e6c812d215",
        "name": "Hatfield",
        "email": "hatfieldblankenship@quotezart.com",
        "role": "user"
    }
}

    This endpoint finds and returns a user object given a user name as a query parameter.

    Example HTTP Request

    GET http://localhost:8000/v1/user?name=Hatfield

    Query Parameters

    Parameter Default Description
    name null The user name must be defined or you'll get an invalid response

    Get a User by Policy ID

    GET 'http://localhost:8000/v1/user/policy?id=4962c4b9-01cc-4cd3-a6c7-0e7b40b55b00'

    The above command returns JSON structured like this:

    {
    "ok": true,
    "msg": "User found successfully",
    "user": {
        "id": "e8fd159b-57c4-4d36-9bd7-a59ca13057bb",
        "name": "Manning",
        "email": "manningblankenship@quotezart.com",
        "role": "admin"
    }
}

    If you're not logged in as an admin, you'll receive the following response:

    {
    "ok": false,
    "msg": "Only admin users can find users by policy ID"
}
    This endpoint is only accessible by admin users so be sure to authenticate with a user that has admin privileges.

    This endpoint finds and returns a user object given a policy ID as a query parameter. If the policy doesn't exist of if there isn't a user associated with that policy, you'll get a 400 status error.

    Remember that you must be logged in as a user with Admin as the role to be able to make this request since it's only accessible by admins.

    Example HTTP Request

    GET http://localhost:8000/v1/user/policy?id=4962c4b9-01cc-4cd3-a6c7-0e7b40b55b00

    Query Parameters

    Parameter Default Description
    id null The policy ID must be defined or you'll get an invalid response

    Errors

    The Insurance API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid or you're lacking an API key which you can get from the login endpoint.
    500 Internal Server Error -- We had a problem with our server. Try again later.
